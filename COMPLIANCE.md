# GitHub 发布合规清单

本清单用于将仓库公开到 GitHub 前的版权、许可证和敏感信息检查。

## 已处理

- 根目录包含 MIT `LICENSE`，版权主体为 `CusOps contributors`。
- `package.json` 和 `package-lock.json` 的根许可证已统一为 `MIT`。
- 已补充第三方依赖许可证摘要：`THIRD_PARTY_LICENSES.md`。
- 已补充资产来源说明：`ASSETS.md`。
- 已补充敏感信息处理说明：`SECURITY.md`。
- README 已标注演示数据为虚构数据。
- 已移除包含疑似第三方浮窗/水印或未引用的截图。
- 已清理 `cusops.md` 中从网页导出残留的原始 HTML UI 标记。

## 发布前仍需人工确认

- 如果项目归属某个公司或个人，请将 `LICENSE` 和 `package.json` 中的 `CusOps contributors` 改为真实权利主体。
- 如果后续加入第三方图片、字体、品牌素材或模板，请先记录来源、许可证和使用范围。
- 如果后续接入真实 LLM、Google Calendar 或支付/短信等服务，请确认对应服务条款、隐私政策和数据处理规则。
- 如果直接推送现有 git 历史，被删除的旧截图仍可从历史提交中取回；公开发布前如果不希望包含这些旧资产，请使用干净的新仓库导入当前文件树，或在确认风险后重写历史。
- 如果仓库曾经提交过真实密钥或客户数据，仅删除当前文件不够；需要吊销密钥并清理 git 历史。
